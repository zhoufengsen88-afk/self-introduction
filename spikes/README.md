# 技术 Spike

本目录存放用于验证高风险技术假设的最小实验代码。Spike 代码强调结论可复现，不代表最终生产目录结构。

## M1 执行顺序

1. `m1_ingestion`：验证知识发布过滤、Markdown 结构化切块、稳定 Chunk ID 和离线评测入口。
2. `m1_embedding`：在同一批 Chunk 和评测问题上比较中文检索效果。
3. `m1_pgvector`：验证可见性过滤、Embedding 版本隔离和精确余弦检索。
4. `m1_sse`：验证增量输出、完成、错误、取消事件和 fetch 流式消费。
5. `m1_llm`：验证 Fake Provider、回答契约、引用、职责边界和拒答；真实模型对比待 API Key。

每项实验都应记录环境、输入、指标、结果和决策，不能只保留“感觉更好”的结论。
