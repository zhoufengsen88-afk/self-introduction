# M5.4 最小可观测性报告

## 1. 目标与范围

M5.4 为面试展示型 Agentic RAG 项目增加足够定位问答失败的非敏感 Trace，不建设完整监控平台、不引入 OpenTelemetry Collector，也不记录访客问题、模型回答或 Prompt 全文。

需要能够回答：

- 请求走了哪个 Route 和 Retrieval Intent；
- 检索并提供了哪些公开 Chunk；
- 最终使用 LLM 还是确定性 Policy；
- 是否拒答以及拒答原因；
- 首 Token 和总耗时；
- 是否发生 Provider、Pipeline 或客户端取消错误。

## 2. 实现

### 2.1 Trace ID

`POST /api/chat` 和 `POST /api/chat/stream` 每次生成一个 32 位十六进制 Trace ID，并通过响应头返回：

```text
X-Trace-ID: 0123456789abcdef0123456789abcdef
```

CORS 已暴露该响应头，浏览器可以读取；即使前端暂不展示，也可以在开发者工具的 Network 面板中关联请求和服务端日志。

### 2.2 Trace 字段

每次完成的问答输出一条 `rag_request_completed` JSON 日志：

```json
{
  "timestamp": "2026-07-18T08:00:00+00:00",
  "level": "INFO",
  "logger": "self_intro_api.rag_trace",
  "message": "rag_request_completed",
  "service": "self-introduction-api",
  "trace_id": "0123456789abcdef0123456789abcdef",
  "route": "knowledge_rag",
  "intent": "challenge",
  "retrieved_chunk_ids": ["skillvar-challenges--核心难点-gitlab-仓库层级关系权限隔离-问题背景--001"],
  "generation_strategy": "llm",
  "refused": false,
  "refusal_reason": null,
  "streaming": true,
  "first_token_ms": 1800.25,
  "total_latency_ms": 4200.75,
  "error_code": null
}
```

`generation_strategy` 可取：

- `llm`：真实模型生成；
- `deterministic`：本地确定性回答器；
- `route_policy`：寒暄、范围说明或安全路由；
- `evidence_policy`：证据不足拒答；
- `grounded_policy`：高风险职责边界的审核回答。

当前错误码包括：

- `llm_provider_error`；
- `rag_pipeline_error`；
- `client_cancelled`。

### 2.3 故障隔离

Trace Sink 使用端口接口与 RAG 业务逻辑隔离。默认实现写入标准输出 JSON；将来需要时可以替换为其他后端。Trace 写入失败只记录 `rag_trace_record_failed`，不会让正常聊天请求失败。

## 3. 隐私边界

Trace 数据结构明确不包含：

- 用户问题或会话历史；
- 模型回答；
- Prompt 和检索 Chunk 正文；
- API Key、Authorization Header、Cookie 或数据库连接串；
- LLM Base URL；
- 引用 excerpt。

只记录公开 Chunk 的稳定 ID，不记录 Chunk 内容。日志可以交给容器或托管平台采集，但仍应按运维数据控制访问和保留时间。

## 4. SSE 与取消

- SSE 首次产生 `delta` 时记录首 Token 延迟。
- 正常完成时记录总延迟和引用对应的 Chunk IDs。
- LLM Provider 失败记录 `llm_provider_error`。
- 流式生成被客户端取消时记录 `client_cancelled`，然后继续传播取消信号，不伪装成正常完成。
- `X-Trace-ID` 在建立 StreamingResponse 时即返回，不依赖生成完成。

## 5. 验证

自动测试覆盖：

- JSON 和 SSE 接口返回格式正确的 `X-Trace-ID`；
- 知识问答记录 route、intent、Chunk IDs 和生成策略；
- 无证据拒答不记录无关 Chunk；
- 普通对话记录 `route_policy` 和首 Token；
- Trace 数据结构不存在问题、回答、Prompt、密钥或 Base URL 字段；
- Trace Sink 故障不影响聊天响应。

## 6. 有意不做

项目按小型面试作品控制范围，当前不实现：

- OpenTelemetry Collector 和分布式 Span；
- Prometheus/Grafana 或商业 LLM Observability SaaS；
- 日志数据库和 Trace 查询 UI；
- 完整 Prompt/回答留存；
- Token 成本统计（当前 Provider 没有返回 Usage）；
- 大规模 P50/P95 指标平台。

下一步直接进入部署准备，并在目标代理/CDN链路验证 SSE、响应头和标准输出日志是否正常。
