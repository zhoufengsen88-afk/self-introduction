# M3.4 语义检索评测报告

## 1. 目标

M3.4 的目标是验证真实 Embedding 是否在“弱关键词、英文、同义改写、多轮指代”问题上优于 M3/M3.2 的工程基线。

旧的 `mvp-v1.jsonl` 更适合验证端到端闭环，但问题大多直接包含“权限、架构、职责、成果”等关键词。因此 M3.4 新增独立数据集：

```text
evals/datasets/m3.4-semantic-v1.jsonl
```

该数据集包含 13 条启用用例，覆盖：

- 英文问题。
- 中文同义改写。
- 弱关键词问题。
- 指标公开边界。
- 职责边界。
- MCP / CLI 分发链路。
- GitLab 权限难点。
- 多轮英文指代。

## 2. 对比对象

| 对象 | 说明 |
| --- | --- |
| memory baseline | 内存 BM25 + 轻量意图重排 |
| hashing pgvector | 本地 Hash Embedding + pgvector + 轻量意图重排 |
| E5 pgvector | `multilingual-e5-small` + pgvector + 轻量意图重排 |

## 3. 结果

| 方案 | Case Count | Document Hit | Chunk Hit | Required Recall | Forbidden Pass |
| --- | ---: | ---: | ---: | ---: | ---: |
| memory baseline | 13 | 61.54% | 61.54% | 42.31% | 100% |
| hashing pgvector | 13 | 92.31% | 53.85% | 56.41% | 100% |
| E5 pgvector | 13 | 100% | 84.62% | 76.92% | 100% |


注意：这些文件路径会被最新 Pipeline 重新生成。M3.5 后的正式对照结果已单独保存为 `evals/results/m3.5-query-understanding-*.json`。

## 4. 结论

E5 在 M3.4 语义压力集上明显优于 Hashing 和内存 BM25：

- 相比内存 BM25，E5 的 Chunk Hit 从 61.54% 提升到 84.62%。
- 相比 Hashing pgvector，E5 的 Chunk Hit 从 53.85% 提升到 84.62%。
- E5 的 Document Hit 达到 100%，说明它能在弱关键词与英文问题下稳定找到正确文档。
- Hashing 的 Document Hit 也较高，但 Chunk Hit 明显较低，说明它能粗略找到文档，却不擅长定位具体证据片段。

因此，M3.3 引入真实 Embedding 是有实际价值的；它不只是“换了个模型”，而是在更难的问题分布上提升了证据召回。

## 5. 未通过用例

M3.4 原始 E5 结果仍有 2 条 Chunk Hit 未通过：

| 用例 | 问题 | 现象 |
| --- | --- | --- |
| `semantic-zh-003` | 这个平台怎么把能力交给自动化工具或命令行使用？ | 命中了相关项目与职责文档，但没有精确命中 MCP / CLI Chunk |
| `semantic-multiturn-001` | What was the hardest part there? | 命中了 Skillvar 相关文档，但多轮英文指代没有稳定定位到权限难点 Chunk |

这说明下一步优化重点不应继续换模型，而应进入查询理解层：

- 中英文意图识别。
- 多轮问题改写。
- 针对 MCP / CLI、权限难点、指标边界等主题的 query expansion。
- 检索后 rerank 或 cross-encoder rerank。

## 6. M3.5 更新

M3.5 已按本报告结论先实现轻量查询理解，而不是继续更换 Embedding 模型。

新增优化包括：

- 权限难点意图：补强 `hardest part`、`inherited access`、私有仓库等表达。
- MCP / CLI 意图：补强“自动化工具、命令行、agents、command line”等表达。
- 检索降级意图：补强 `semantic search channel unavailable` 到 ChromaDB / BM25-only 降级证据的映射。
- 意图优先级修正：职责和团队边界优先于 MCP / CLI，避免主评测集回归。

同一语义压力集在 M3.5 后结果如下：

| 方案 | Case Count | Document Hit | Chunk Hit | Required Recall | Forbidden Pass |
| --- | ---: | ---: | ---: | ---: | ---: |
| memory baseline | 13 | 76.92% | 76.92% | 61.54% | 100% |
| hashing pgvector | 13 | 100% | 84.62% | 76.92% | 100% |
| E5 pgvector | 13 | 100% | 100% | 100% | 100% |

详细报告见 [M3.5 查询理解优化报告](./m3.5-query-understanding-report.md)。

## 7. 边界

- 当前数据集仍然很小，只有 13 条语义压力用例。
- 当前回答仍由确定性证据组织器生成，不代表真实 LLM 生成质量。
- 当前评测重点是证据检索，不是自然语言表达。
- 后续应继续扩充英文、同义表达、跨文档和无证据用例。
