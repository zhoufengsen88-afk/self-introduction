# M3.3 真实 Embedding Provider 报告

## 1. 目标

M3.3 的目标是把 M3.2 的本地 Hash Embedding 工程基线替换为真实语义 Embedding Provider，并保留可比较的评测结果。

本阶段验证的是：

- 真实模型依赖可以在正式项目环境中安装和运行。
- Embedding Provider 可以通过统一接口接入。
- 模型名、revision、维度和前缀策略可以被版本化记录。
- E5 向量可以进入 PostgreSQL + pgvector。
- DB 检索结果可以复用同一套 RAG 回答、引用和拒答逻辑。

## 2. 当前模型

| 项目 | 内容 |
| --- | --- |
| Provider | `SentenceTransformerEmbeddingProvider` |
| 模型 | `intfloat/multilingual-e5-small` |
| revision | `614241f622f53c4eeff9890bdc4f31cfecc418b3` |
| 维度 | 384 |
| query prefix | `query: ` |
| passage prefix | `passage: ` |
| 距离 | L2 normalize 后使用 pgvector 余弦距离 |
| 依赖 | `sentence-transformers` |

## 3. 命令

```bash
make ingest-db-e5
make ask-db-e5 QUESTION="你在 Skillvar 中具体负责什么？"
make eval-db-e5
```

首次运行会从 Hugging Face 下载模型。未登录时可以运行，但可能遇到下载限流或偶发连接错误；可配置 `HF_TOKEN` 提高稳定性。

## 4. 本地验证结果

临时 PostgreSQL 18 + pgvector 实例验证：

```text
documents: 5
chunks: 38
embeddings: 38
embedding_model: multilingual-e5-small
embedding_revision: 614241f622f53c4eeff9890bdc4f31cfecc418b3
embedding_dimension: 384
```

E5 版问答能够返回预期引用，例如职责问题的首个引用为：

```text
skillvar-responsibilities--已确认的个人实现范围-工程问题处理--001
```

## 6. 解释边界

- 当前评测集只有 16 条启用用例，且 Skillvar 文档结构清晰，因此 E5 和 Hashing baseline 都能达到 100%。
- 这说明当前 RAG 工程链路稳定，但不能说明语义检索已经充分泛化。
- 下一步应扩充评测集，加入同义表达、英文问题、跨文档问题、无关键词问题和更难的多轮指代。
- 真实 LLM Provider 尚未接入，当前回答仍由确定性证据组织器生成。
